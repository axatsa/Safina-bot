## Роли и права

| Действие | Сотрудник (бот) | Сафина (веб) | CFO | CEO |
|---|---|---|---|---|
| Заполнить и отправить заявку на расходы | ✅ | ✅ | — | — |
| Заполнить и отправить служебную записку (бланк) | ✅ | — | — | — |
| Заполнить и отправить заявление на возврат | ✅ | — | — | — |
| Видеть свои заявки | ✅ | — | — | — |
| Скачать .docx / .xlsx | ❌ | ✅ | ✅ | ✅ |
| Менять статус в канбан | ❌ | ✅ | ✅ | ✅ |
| Отправить на согласование CFO | ❌ | ✅ | — | — |
| Одобрить / отклонить (CFO) | ❌ | — | ✅ | — |
| Одобрить / отклонить (CEO) | ❌ | — | — | ✅ |

**Главный принцип:** сотрудник через бот только **отправляет** → Сафина получает, проверяет, печатает, двигает дальше.

## ЧАСТЬ 1 — Бот: что меняется в воронках бланков

### Текущая проблема

В `blank_wizard.py` в конце воронки стоит кнопка `📥 Скачать бланк` и бот
отправляет сотруднику `.docx` файл. Это неправильно — сотрудник не должен
получать файл.

### Что должно быть

Сотрудник заполняет данные → видит экран подтверждения → нажимает
**"✅ Отправить"** → данные сохраняются в БД → Сафина получает
уведомление в Telegram → сотрудник видит "Заявка отправлена".

Никакого `.docx` сотруднику в боте.

### Новый финальный экран в боте (вместо скачивания)

**Служебная записка:**
```
📋 Проверьте данные

Бланк:     LAND
ФИО:       Баходиров Баходир
Должность: Менеджер отдела LAND
Цель:      Закупка канцтоваров

Позиции:
  1. Ручка синяя — 10 × 5 000 UZS
  2. Бумага А4   —  2 × 35 000 UZS

Итого: 120 000 UZS

[ ✅ Отправить Сафине ]  [ ◀️ Назад ]
```

После нажатия "Отправить":
```
✅ Служебная записка отправлена!

Сафина получила уведомление и
обработает вашу заявку.
```

**Заявление на возврат** — то же самое, кнопка "✅ Отправить Сафине",
после — подтверждение.

## ЧАСТЬ 2 — Что сохраняется в БД при отправке бланка

При нажатии "Отправить Сафине" создаётся запись в `expense_requests`:

```python
ExpenseRequest(
    request_type   = "blank",         # новый тип, отличает от "expense" и "refund"
    template       = "land",          # ключ шаблона
    purpose        = "Закупка...",
    items          = [...],           # список позиций
    total_amount   = 120000,
    currency       = "UZS",
    status         = "request",       # стандартный начальный статус
    created_by_id  = user.id,
    created_by     = "Баходиров Баходир",
    created_by_position = "Менеджер отдела LAND",
    project_id     = user.projects[0].id,  # если есть
)
```

Для заявления на возврат — `request_type = "blank_refund"`, данные
клиента сохраняются в `refund_data` (уже есть это поле).

## ЧАСТЬ 4 — Веб-приложение (что меняется)

### Страница инвестиций (канбан)

Бланки и заявления на возврат должны появляться в канбан-доске Сафины
как отдельные карточки с пометкой типа:

```
[📋 LAND-42]  Баходиров Баходир  120 000 UZS
[🧾 SCH-43]   Баходиров Баходир  1 500 000 сум
```

В карточке показывать `request_type` иконкой:
- `expense` → 💰
- `refund` → 🔄
- `blank` → 📋
- `blank_refund` → 🧾

### Детальная страница бланка

На странице `ExpenseDetail` для бланков показывать:
- Все введённые данные
- Кнопки скачать `.docx` — **только для ролей admin, senior_financier, ceo**
- Кнопка "Отправить CFO" — **только для admin**

### Скачивание — только для Сафины и выше

В `ExpenseDetail.tsx` кнопки скачать обернуть в проверку роли:

```tsx
{/* Показывать только admin, senior_financier, ceo */}
{canDownload(currentUser.role) && (
  <Button onClick={() => downloadBlank(expense.id)}>
    📄 Скачать .docx
  </Button>
)}
```

В `rbac.ts` добавить:
```typescript
export const canDownload = (role: string) =>
  ["admin", "senior_financier", "ceo"].includes(role);
```

На бэкенде эндпоинт скачивания тоже проверяет роль:
```python
@router.get("/{expense_id}/export-blank-docx")
def export_blank(
    expense_id: str,
    current_user = Depends(auth.get_current_user)
):
    if not auth.is_admin(current_user):
        raise HTTPException(status_code=403,
            detail="Только администратор может скачивать документы")
    # ... генерация файла
```

## ЧАСТЬ 5 — Статусы для бланков в канбане

Бланки проходят упрощённый путь по сравнению с инвестициями:

```
request → review → confirmed
                ↘ declined
```

Сафина в канбане двигает карточку:
- `request` → получена, ждёт обработки
- `review` → Сафина взяла в работу (распечатала)
- `confirmed` → подписана, закрыта
- `declined` → отклонена (с комментарием)

Статусы `pending_senior`, `pending_ceo`, `approved_senior` и т.д. для
бланков **не используются** — они не идут на согласование CFO/CEO,
это внутренние документы отдела.

## ЧАСТЬ 6 — Что убрать из кода

| Где | Что убрать |
|---|---|
| `blank_wizard.py` | Хендлер `download_blank`, кнопку `📥 Скачать бланк` |
| `blank_wizard.py` | `F.text == "📥 Скачать бланк"` |
| Финальный экран в боте | Кнопку скачать → заменить на `✅ Отправить Сафине` |
| `BlankForm.tsx` (WebApp) | Кнопку "Скачать" → заменить на "Отправить" |

## Итоговый поток

```
Сотрудник в боте:
  Заполнил данные → "✅ Отправить Сафине" → запись в БД → уведомление Сафине
  Видит: "✅ Отправлено!"
  НЕ получает: .docx файл

Сафина в Telegram:
  Получает уведомление с данными + кнопка [📄 Скачать .docx]
  Нажимает → получает файл прямо в Telegram

Сафина в веб-панели:
  Видит карточку в канбане
  Может скачать .docx, сменить статус, добавить комментарий
  Может отправить инвестицию (не бланк) дальше к CFO

Сотрудник в веб-панели (если есть доступ):
  Видит только свои заявки
  Кнопок скачать НЕТ
```
