## ⚠️ ЕСЛИ НЕ ВСЕ ГАЛОЧКИ

### GitLab CI не прошел (красный)

**Действия:**
1. Открыть красный job в GitLab
2. Скопировать последние 20 строк логов
3. Прочитать ошибку
4. Если "no such file" → вернуться к исправлению файлов
5. Если "authentication" → проверить GitLab variables (CI/CD Settings)

### Контейнер в статусе Restarting

**Действия:**
1. Посмотреть логи: `docker-compose logs app --tail=100`
2. Найти строку с ошибкой (обычно в конце)
3. Если "ModuleNotFoundError" → переходить к ПЛАНУ B
4. Если другая ошибка → записать и искать решение

### Сайт все еще 404

**Действия:**
1. Проверить что контейнер Up (не Restarting)
2. Проверить логи контейнера (должен быть nginx)
3. Проверить .env файл на сервере (IMAGE_URL правильный?)
4. Попробовать ручной перезапуск:
   ```bash
   cd /home/finance/frontend/main
   docker-compose down
   docker-compose pull app
   docker-compose up -d app
   ```
5. Если не помогло → ПЛАН B
