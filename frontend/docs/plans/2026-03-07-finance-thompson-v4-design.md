# Design: Обновление системы Finance Thompson (v4)

## 1. Архитектура (Микросервисы)
Система будет разделена на независимые микросервисы для обеспечения масштабируемости:

1.  **Core API (FastAPI)**: Основной REST API для обработки бизнес-логики (CRUD заявок, авторизация, статистика).
2.  **Telegram Bot Service (Aiogram 3)**: Выделенный сервис для взаимодействия с ботом (отправка заявок финансового контролера, получение команд "Одобрить/Отклонить").
3.  **Notification Service (WebSockets/SSE + Redis)**: Сервис реального времени для push-уведомлений Сафине в Web-App.
4.  **Worker Service (Celery/RQ)**: Фоновый обработчик для тяжелых задач (генерация Excel-смет, массовые выгрузки).
5.  **Базы данных**: PostgreSQL (основные данные) и Redis (брокер сообщений Pub/Sub и кэш).

---

## 2. Рабочий процесс (Двухуровневое подтверждение)

1.  **Сафина**: Проверяет заявку в Web-App. Для крупных сумм нажимает кнопку «Отправить на согласование».
2.  **Core API**: Меняет статус заявки на `PENDING_SENIOR` и публикует событие в брокер сообщений (Redis).
3.  **Bot Service**: Ловит событие и отправляет сообщение Старшему финансисту в Telegram с деталями заявки и кнопками: [📄 Скачать смету], [✅ Одобрить], [❌ Отклонить].
4.  **Старший финансист**:
    *   При нажатии [📄 Скачать смету]: Bot Service отправляет задачу в **Worker Service**. Worker генерирует Excel и возвращает файл в бот.
    *   При нажатии [✅ Одобрить] / [❌ Отклонить]: Bot Service отправляет запрос в Core API.
5.  **Core API**: Обновляет статус заявки (`APPROVED_SENIOR` / `REJECTED_SENIOR`) и публикует событие об изменении статуса.
6.  **Notification Service**: Ловит событие статуса и мгновенно пушит Toast-уведомление в React-клиент Сафины.

---

## 3. Генерация отчетов (Excel)

*   **Одиночные сметы (.xlsx)**: Строятся на основе предоставленного шаблона (Цвета, рамки, желтая шапка). Используются библиотеки `openpyxl` или `xlsxwriter` для точного позиционирования данных (ID, дата, проект, сумма и т.д.) в нужные ячейки.
*   **Массовая выгрузка**: Во вкладке "Статистика" будет кнопка выгрузки архива за выбранный период. Будет генерироваться сводная таблица со всеми заявками и итоговыми суммами.

---

## 4. Пользовательский интерфейс (Frontend)

*   **Карточка заявки (Детали)**: Добавление новой кнопки `[Отправить на согласование]` (появляется только если статус позволяет это сделать).
*   **Уведомления**: Интеграция компонента Toast (из Shadcn/UI) с WebSocket/SSE клиентом для приема уведомлений от сервера.
*   **Вкладка «Статистика»**:
    *   Реализация фильтров по времени (Месяц, 3 мес, 6 мес, Год).
    *   Отображение виджетов: Общие суммы (Школа/Садик), Процент конверсии (Одобрено/Отклонено Старшим фин.).
    *   Интеграция графиков (например, `recharts` или `chart.js`): динамика по месяцам (Line/Bar chart) и круговые диаграммы долей филиалов (Pie chart).

---

## 5. Инфраструктура и Безопасность

*   **Учетные записи**: Автоматическое создание аккаунтов (логин/пароль) для Сафины и Старшего финансиста (на случай необходимости входа в админ-панель) при инициализации БД.
*   **Миграция**: Обновление настроек CORS в `main.py` и переменных окружения (`.env`) для поддержки нового домена приложения вместо старого Cloudflare-домена. Убедиться, что Webhook бота ссылается на правильный URL.
