# Thompson Finance | Safina Expense Tracker

Профессиональная система управления расходами с веб-интерфейсом и интегрированным Telegram-ботом.

---

## 🏗 Архитектура проекта

Проект разработан как Full-stack приложение, разделенное на независимые части: **Frontend** (современное SPA на React) и **Backend** (высокопроизводительный API на FastAPI).

### 🖥 Frontend (`/frontend`)
Построен на самом современном стеке технологий для обеспечения скорости и премиального дизайна.

*   **Стек**: React + Vite + TypeScript.
*   **Стилизация**: Tailwind CSS с использованием системы компонентов **Shadcn/UI**.
*   **Дизайн**: Использована концепция *Glassmorphism* (стеклянные элементы), плавные микро-анимации (`framer-motion`) и кастомная типографика (Inter).
*   **Состояние и Данные**:
    *   `@tanstack/react-query`: Мощное кэширование данных и управление запросами.
    *   `src/lib/store.ts`: Централизованный слой взаимодействия с API. Все запросы к бэкенду проходят здесь.
*   **Структура папок**:
    *   `src/components/ui/`: Базовые UI-компоненты (кнопки, вводы, диалоги).
    *   `src/components/`: Сложные компоненты, такие как `ExpenseCard` (карточка расхода).
    *   `src/pages/`: Основные экраны приложения (Dashboard, Login, ExpenseDetail, Team, Projects).
    *   `src/hooks/`: Кастомные хуки для логики.
*   **Конфигурация**: Файл `.env` содержит `VITE_APP_API_URL` для связи с бэкендом.

### ⚙️ Backend (`/backend`)
Обеспечивает безопасность, работу с базой данных и интеграцию с Telegram.

*   **Стек**: Python + FastAPI + SQLAlchemy.
*   **Основные модули (`app/`)**:
    *   `main.py`: Точка входа. Управляет CORS, маршрутами и жизненным циклом (Lifespan), запуская бота вместе с сервером.
    *   `app/db/`: Описание схем данных (models) и CRUD-операций. Здесь же логика генерации Request ID (например, TS-1).
    *   `app/api/`: Обработка запросов (авторизация, расходы, статистика).
    *   `app/services/bot/`: Полная логика Telegram-бота (Aiogram 3.x).
*   **Экспорт данных**: Прямая генерация Excel-отчетов (`pandas`) и Word-смет (`docxtpl`).
*   **Устойчивость**:
    *   **Global Exception Handler**: Все ошибки API возвращаются в едином JSON-формате.
    *   **Health Check**: Эндпоинт `/api/health` для мониторинга состояния сервера.
*   **Тестирование**: Встроенная база тестов на `pytest` (папка `tests/`).
*   **Шаблоны**: `template.docx` — визуальный шаблон вашей сметы, который заполняется данными автоматически.

## 🚀 Развертывание (Docker)

Проект полностью контейнеризирован и готов к продакшну.

### Переменные окружения (`.env`):
Создайте файл `.env` в корневом каталоге (или в папке `backend`):
```env
BOT_TOKEN=...           # Токен от @BotFather
ADMIN_LOGIN=safina      # Логин администратора
ADMIN_PASSWORD=...      # Пароль администратора
WEB_FORM_BASE_URL=...   # URL фронтенда для ссылок из бота
POSTGRES_...            # (Опционально) Настройки БД
```

### Команды для управления:
*   **Запуск**: `docker-compose up -d --build`
*   **Обновление**: `git pull && docker-compose up -d --build`
*   **Удаление лишних контейнеров**: `docker-compose up -d --remove-orphans`
*   **Логи бэкенда**: `docker logs -f finance-backend-main`

## 🔄 Работа с Git (GitLab & GitHub)

Проект настроен для одновременной работы с рабочим **GitLab** ( Thompson Finance) и личным **GitHub** (для портфолио).

### 1. GitLab (Рабочий — деплой на сервер)
Для обновления кода на сервере нужно заходить в соответствующие папки:

*   **Backend**: `cd backend` -> `git push origin main`
*   **Frontend**: `cd frontend` -> `git push origin main`

### 2. GitHub (Портфолио — весь проект)
Для обновления вашего личного репозитория на GitHub нужно заходить в **корень проекта**:

*   **Корень**: `cd ..` (выйти из папок) -> `git push github main`

> [!IMPORTANT]
> GitHub-репозиторий хранит весь проект целиком одним монолитом, в то время как GitLab-репозитории разделены для удобства деплоя через Docker.

## 📈 Ключевые функции
1.  **Умный расчет**: Поддержка полей количества и цены для автоматического формирования итогов.
2.  **Автоматизированная система бланков**: API для создания заявок и интеграция с `docxtpl` для генерации документов.
3.  **Канбан-доска (API)**: Поддержка состояний и переходов для заявок на возврат.
4.  **Мгновенные уведомления**: Интеграция с Telegram Bot API (Aiogram 3) для взаимодействия с админами и финансистами.
5.  **Гибкие сметы**: Генерация профессиональных смет в Word и Excel одним нажатием кнопки.
6.  **Безопасность**: Ролевая модель доступа и защита API через JWT токены.
