# Руководство по локальному запуску (без Docker)

Это руководство поможет вам запустить проект **Safina Expense Tracker** на локальной машине для разработки.

## 📋 Предварительные требования

Перед началом убедитесь, что у вас установлены:
- **Python 3.10+**
- **Node.js 18+** и **npm**
- **Git**

---

## ⚙️ 1. Настройка бэкенда (FastAPI)

1. Перейдите в директорию бэкенда:
   ```powershell
   cd backend
   ```

2. Создайте виртуальное окружение:
   ```powershell
   python -m venv venv
   ```

3. Активируйте виртуальное окружение:
   - В Windows: `.\venv\Scripts\activate`
   - В Linux/macOS: `source venv/bin/activate`

4. Установите зависимости:
   ```powershell
   pip install -r requirements.txt
   ```

5. Настройте переменные окружения:
   Скопируйте `.env.example` в новый файл `.env` и заполните его:
   ```env
   DATABASE_URL=sqlite:///./safina.db
   SECRET_KEY=ваш_секретный_ключ
   BOT_TOKEN=ваш_токен_телеграм_бота
   ADMIN_LOGIN=safina
   ADMIN_PASSWORD=ваш_пароль
   ```

6. Запустите сервер:
   ```powershell
   python main.py
   ```
   *Сервер будет доступен по адресу: http://localhost:8000*

---

## 🎨 2. Настройка фронтенда (React + Vite)

1. Перейдите в директорию фронтенда:
   ```powershell
   cd frontend
   ```

2. Установите зависимости:
   ```powershell
   npm install
   ```

3. Настройте файл `.env`:
   Создайте файл `frontend/.env` (если его нет) и укажите URL бэкенда:
   ```env
   VITE_APP_API_URL=http://localhost:8000
   ```

4. Запустите фронтенд в режиме разработки:
   ```powershell
   npm run dev
   ```
   *Приложение будет доступно по адресу: http://localhost:5173 (или другом, указанном в консоли)*

---

## 🤖 3. Настройка Telegram-бота

Бот запускается автоматически вместе с бэкендом, если в `.env` указан `BOT_TOKEN`.
- Убедитесь, что у вас есть токен от [@BotFather](https://t.me/BotFather).
- Установите веб-хуки или убедитесь, что бот работает в режиме Polling (по умолчанию).

---

## 🧪 4. Запуск тестов

Для проверки работоспособности бэкенда:
```powershell
python -m pytest tests/test_api.py
```
*Тесты используют изолированную базу данных в памяти (SQLite), не затрагивая ваш основной файл `safina.db`.*

---

## 🛠 Устранение неполадок

- **База данных**: При первом запуске бэкенд автоматически создаст файл `safina.db` (SQLite) в папке `backend`.
- **CORS**: Бэкенд настроен на прием запросов с `http://localhost:5173` и других стандартных портов Vite.
- **Порты**: Убедитесь, что порты 8000 (бэкенд) и 5173 (фронтенд) не заняты другими процессами.
