# ТЗ: Система скачивания бланков — Thompson Finance Bot

---

## Контекст

В проекте уже реализована генерация `.docx` через библиотеку **docxtpl**:
- шаблон содержит плейсхолдеры `{{ переменная }}`
- функция `generate_docx(template_path, data)` подставляет данные и возвращает `BytesIO`
- эта же функция используется для генерации смет и заявлений на возврат

**Новые бланки работают по той же схеме.** Ничего нового изобретать не нужно.

---

## Шаблоны

Всего **5 шаблонов** `.docx`:

| Файл | Тип | Используется для |
|---|---|---|
| `land.docx` | Служебная записка | Проект LAND (садик) |
| `ls.docx` | Служебная записка | Проект ЛС |
| `management.docx` | Служебная записка | Проект Management |
| `school.docx` | Служебная записка | Проект School |
| `refund.docx` | Заявление на возврат | Все проекты |

Все 4 служебных записки **идентичны по структуре** — различаются только визуально (шапка/логотип). Плейсхолдеры одинаковые.

Хранятся в: `backend/app/services/docx/templates/`

## Шаг 0 — Подготовка шаблонов (разовая работа)

Нужно один раз открыть каждый `.docx` в Word и вставить плейсхолдеры `{{ }}` вместо захардкоженных данных.

### Служебная записка — плейсхолдеры

| Место в документе | Плейсхолдер | Пример |
|---|---|---|
| Шапка "от..." | `{{ sender_position }} {{ sender_name }}` | `Менеджер отдела LAND Баходиров Б.` |
| Цель расхода | `{{ purpose }}` | `Закупка канцтоваров` |
| Строка таблицы (цикл) | `{% for item in items %}` | — |
| — Наименование | `{{ item.name }}` | `Ручка синяя` |
| — Количество | `{{ item.qty }}` | `10` |
| — Сумма | `{{ item.amount }} {{ item.currency }}` | `5 000 UZS` |
| `{% endfor %}` | конец цикла | — |
| Итого | `{{ total_amount }} {{ currency }}` | `120 000 UZS` |
| Подпись (должность) | `{{ sender_position }}` | `Менеджер отдела LAND` |
| Дата | `{{ date }}` | `14.03.2026` |
| ФИО под подписью | `{{ sender_name_short }}` | `Баходиров Б.` |

> `sender_name_short` генерируется автоматически из полного ФИО: `"Баходиров Баходир"` → `"Баходиров Б."`

### Заявление на возврат — плейсхолдеры

| Место | Плейсхолдер |
|---|---|
| ФИО клиента | `{{ client_name }}` |
| Паспорт | `{{ passport_series }} {{ passport_number }}`, `{{ passport_issued_by }}`, `{{ passport_date }}` |
| Телефон | `{{ phone }}` |
| Договор | `{{ contract_number }}`, `{{ contract_date }}` |
| Причина — чекбоксы | `{{ reason_pereezd }}`, `{{ reason_grafik }}`, `{{ reason_ozhidaniy }}`, `{{ reason_trudnosti }}`, `{{ reason_lichnye }}`, `{{ reason_drugoe }}` — каждый принимает значение `☑` или `□` |
| Если "Другое" | `{{ reason_drugoe_text }}` |
| Сумма | `{{ amount }}`, `{{ amount_words }}` |
| Владелец карты | `{{ card_holder }}` |
| Номер карты | `{{ card_number }}` |
| Транзитный счёт | `{{ transit_account }}` |
| ИИН банка | `{{ bank_iin }}` |
| МФО банка | `{{ bank_mfo }}` |
| Наименование банка | `{{ bank_name }}` |
| Дата заявления | `{{ date }}` |
| ФИО директора | `{{ director_name }}` |
| Филиал | `{{ branch }}` |

## Шаг 1 — Откуда берутся данные

Пользователь уже авторизован. В модели `TeamMember` есть:

```
first_name, last_name  →  sender_name (собирается автоматически)
position               →  sender_position (берётся автоматически)
branch                 →  branch (берётся автоматически)
```

**Пользователю нужно ввести только:**
- Для служебной записки: **цель расхода + позиции** (2-7 шагов)
- Для заявления на возврат: **данные клиента + реквизиты** (длинная форма)

## Шаг 4 — Вариант Б: Web форма (WebApp)

При нажатии "🌐 Открыть Web форму" бот отправляет кнопку `WebAppInfo`:


Параметр `type` определяет какой шаблон и какие поля показывать.

## Шаг 5 — API эндпоинт

Один универсальный эндпоинт для генерации всех 5 бланков. Добавляется в `backend/app/api/`.

### Запрос

```
POST /api/blank/generate
Authorization: Bearer <token>

Content-Type: application/json
```

**Для служебной записки:**
```json
{
  "template": "land",
  "sender_name": "Баходиров Баходир",
  "sender_position": "Менеджер отдела LAND",
  "purpose": "Закупка канцтоваров",
  "items": [
    { "name": "Ручка синяя", "qty": 10, "amount": 5000, "currency": "UZS" },
    { "name": "Бумага А4",   "qty":  2, "amount": 35000, "currency": "UZS" }
  ],
  "total_amount": 120000,
  "currency": "UZS",
  "date": "14.03.2026"
}
```

**Для заявления на возврат:**
```json
{
  "template": "refund",
  "client_name": "Иванов Иван Иванович",
  "passport_series": "AA",
  "passport_number": "1234567",
  "passport_issued_by": "ГУВД Юнусабадского района",
  "passport_date": "01.01.2020",
  "phone": "+998901234567",
  "contract_number": "12345",
  "contract_date": "15.01.2025",
  "reason": "Переезд",
  "reason_other": "",
  "amount": 1500000,
  "amount_words": "Один миллион пятьсот тысяч",
  "card_holder": "Иванов Иван Иванович",
  "card_number": "8600123456789012",
  "transit_account": "",
  "bank_iin": "",
  "bank_mfo": "",
  "bank_name": "АО Капиталбанк",
  "date": "14.03.2026"
}
```

### Ответ

```
HTTP 200 OK
Content-Type: application/vnd.openxmlformats-officedocument.wordprocessingml.document
Content-Disposition: attachment; filename="blank_land_14032026.docx"

<binary .docx content>
```

### Логика бэкенда

```
1. Определить путь к шаблону по полю "template"
2. Вызвать существующий generate_docx(template_path, data)
3. Вернуть BytesIO как файловый ответ
```

## Что нужно реализовать

### Backend (`backend/app/`)

| Файл | Действие |
|---|---|
| `services/docx/templates/` | Положить 5 подготовленных `.docx` с плейсхолдерами |
| `services/docx/generator.py` | Уже готов, ничего не менять |
| `api/blanks.py` | Новый файл — эндпоинт `POST /api/blank/generate` |
| `main.py` / роутер | Подключить новый роутер `blanks` |

### Bot (`backend/app/services/bot/`)

| Файл | Действие |
|---|---|
| `keyboards.py` | Добавить 5 кнопок в `get_main_kb()` |
| `states.py` | Добавить `BlankWizard` и `RefundBlankWizard` |
| `handlers/blank_wizard.py` | Новый — FSM для служебных записок |
| `handlers/refund_blank_wizard.py` | Новый — FSM для заявления на возврат |
| `handlers/__init__.py` | Зарегистрировать 2 новых роутера |

### Frontend (`frontend/src/`)

| Файл | Действие |
|---|---|
| `pages/BlankForm.tsx` | Новая страница `/blank?type=...` |
| `lib/services/blanks.ts` | `generateBlank(type, data)` → POST → скачать файл |
| `App.tsx` | Добавить маршрут `/blank` |
