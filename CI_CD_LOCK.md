## ❗ Главные правила (Почему всё ломается):

1. **Версия Docker (image: docker:latest)**: 
   - Сервер требует минимум API 1.44. Старые версии (вроде `docker:24-dind`) выдают ошибку `client version 1.43 is too old`. 
   - **РЕШЕНИЕ**: Всегда используйте `image: docker:latest` и `services: - docker:dind`.

2. **Имя сервиса (app)**:
   - Внутри серверных папок `backend/main` и `frontend/main` файлы `docker-compose.yml` называют основной сервис просто **`app`**, а не `backend` или `frontend`.
   - **РЕШЕНИЕ**: Команды деплоя должны быть: `docker-compose pull app` и `docker-compose up -d app`. Иначе будет ошибка `No such service`.

3. **Теги (tags: - docker)**:
   - Ваш GitLab Runner `thompson` настроен без тегов. Если добавить `tags: - docker`, пайплайн застрянет в "Pending" (желтый кружок) навсегда.
   - **РЕШЕНИЕ**: Оставляйте секцию тегов пустой или удаляйте её.

4. **Ошибка ContainerConfig**:
   - На сервере стоит старый `docker-compose` (1.29.2). Он падает при попытке обновить контейнеры с новыми метаданных.
   - **РЕШЕНИЕ**: Всегда добавляйте флаг `--no-deps` при запуске: `docker-compose up -d --no-deps app`. Это обновляет только само приложение, не трогая базу данных.

5. **Фикс 404 (Порты)**:
   - Фронтенд (Nginx) внутри контейнера слушает порт **80**.
   - **РЕШЕНИЕ**: В корневом `docker-compose.yml` проброс должен быть `3000:80`. Если поставить `3000:3000`, сайт выдаст 404.

6. **Структура пайплайна (Три этапа)**:
   - Для визуального контроля всегда используем 3 этапа: 🔨 build, 🧪 test, 🚀 deploy.
